# 项目核心
**装饰成上班时候查阅文档和AI对话，实际是看小说的本地渲染网站，装饰细节的逼真度是本项目的核心竞争力。**
**项目里不要直接出现伪装这个词，可以考虑用装饰、氛围感之类的词语**
**项目里用BOSS代替老板这个词**
**项目里用作品代替小说这个词**
**界面风格高仿中国大陆的网站时候，避免出现该网站的logo和名称，可以用其他词语代替，例如：经典博客**

# 项目需求与最终功能记录 (Requirement & Feature Record)
> 💡 *说明：文件已精简，重点保留防伪原逻辑、交互特色以及底层的特殊实现机制，新功能增添请以最新覆盖为准。*

## 1. 核心阅读与装饰机制
- **阅读额度卡控**：普通用户限制 **30万字** 免费额度。满 90% 轻提示预警，100% 强制弹 Pro 激活窗。校验为本地静默 Token 校验。
- **深层极客皮肤**：包含 VS Code, 终端, IDEA, 掘金, Stack Overflow 等皮肤。Gemini、MDN、ChatGPT 为免费版，其余按 Pro 系统划分。
  - **Beta 标志自定义**：支持通过 `betaText` 自定义标签文字（如“限免”）。
  - **排序优化**：将“经典博客1”和“经典文档1”置于 Beta 列表首位，并标注为“限免”。
- **经典博客1风格（classic_blog1）**：高度还原博客园（cnblogs）经典布局，包含顶部蓝色导航栏、博客头部（含博主名和副标题）、二级导航栏（含统计数据）、左侧博客侧边栏（公告/日历/搜索/常用链接/随笔分类/随笔档案）和主内容区。侧边栏的"随笔分类"实际映射为作品列表，支持导航。**现已增加翻页能力（Pagination）和内部滚动条（Scrollbar），支持在作品数量较多时进行分页浏览并保持侧边栏布局稳固。**该风格通过独立的 `ClassicBlog1Layout.vue` 组件实现，完全替换标准的 Sidebar + ReaderScreen 布局。隐藏AI聊天元素（头像、对话气泡样式、操作按钮），将内容呈现为博客文章形式。
- **高度定制品牌**：支持自定义全局项目名（顶栏左侧）。
  - **快速项目更名**：预置了 `FindDeep`, `DeepReader`, `DeepWork` 等项目名，支持一键切换。
  - **老板键整合**：双击项目名文字即可极速触发老板键。
  - **防破绽**：点击不存在的菜单项显示极客“知识荒原”占位，小屏下自动折叠标题防重叠。
  - **二次排版内容装饰 (规划/讨论)**：对小说自身内容格式进行二次打散和装饰（如转换为日志格式、附加代码块环境、替换小说特有词汇、转为 Markdown 列表排版等），从阅读内容本身的形态上防侦破，降低被老板轻易察觉的风险。
  - **内容辅助块**：在“深度排版内容辅助”中支持多选插入“功能内容块”（如鉴权存根、系统日志、开源证书、数据结构定义）。这些块可以成对出现在渲染作品的正文前后，且独立于正文的 Obfuscation Code-Block 渲染，极大增强了界面充实度和工作氛围真实感。

## 2. 界面交互与体验打磨
- **非阻塞式弹窗**：全局彻底禁用 `alert/confirm`，采用基于状态机带防呆机制的 `<ConfirmModal />` 组件。
- **透明沉浸预览**：在设置皮肤时触发“下沉式预览”（透明度降至0.4并微缩，1.5秒后归位），更直观感知背景。
- **精细化列表管理**：左侧列表支持时间轴分组（实际为扁平渲染），放弃破坏沉浸感的“已置顶”分类标题，改由右侧长驻的 `图钉` Icon 标识置顶位（悬浮交互让出操作栏）；列表拥有纯本地无延迟的强过滤标题搜索；TOC阅读目录自带关键字过滤。
- **侧边栏与视觉对齐**：
  - **离线图标**：剔除外网 CDN，使用 `unplugin-icons` 引入内联 SVG，断网环境同样无瑕疵。
  - **网格系统**：使用 12px 栅格与精准计算偏移，侧边栏左侧全员实现了24px准线笔直对齐。选中态色块低调处理。
  - **个人资料**：支持昵称修改、Base64 头像上传及调色盘控制的自定义渐变头像背景色。
  - **隐藏设置入口 (针对装饰皮肤)**：在切换到除 Gemini 和 ChatGPT 以外的皮肤（如 VS Code, Terminal 等）后，侧边栏底部设置区自动隐藏，改为左下角一个极小且收缩的悬浮按钮。该按钮仅在鼠标 hover 时展开，极大降低了界面出现“设置”字样的视觉突兀感。

## 3. 高级功能体验 (进阶项)
- **多维度防范老板键**：除了可自定义最多3组快捷键，还支持 **鼠标左键连击三下** 迅速逃生。支持配置切换后对话的流式输出开关，并允许自定义展示模板、随机已有AI对话或指定已有AI对话，极大提升防伪真实度。
- **成就彩蛋系统**：完成首次老板键逃生即可解锁“老板关怀”成就，并随之解锁隐藏进阶功能。
- **极客反馈快照**：分类反馈 Bug / 需求时，自带系统案发现场快照收集（浏览器、分辨率、当前使用的皮肤等）。
- **免责合规声明**：高级设置页底部加入严格免责，明示此应用仅为本地解析工具，不涉及线上存储和分发以防法律风险。

## 4. AI 逼真对话与大模型深度融合 (核心难点)
- **内容池（Fake Content Pools）**：预置了 **IT研发类**、**产品设计类** 和 **职场通用类** 三大文案池，支持定时刷新或手动随机。**新增“列表条数随长度增长”功能**，可根据当前对话轮数动态增加侧边栏条数，极度还原“工作越聊越多”的紧迫感。
- **高仿真示例数据 (瀑布流)**：内置三套语料库（IT / 设计 / 通用）。可定义对话条数与多轮次，隐去传统分页，采用真实 AI 的一屏连贯流式渲染（包含动态追问提取）。IT 对话支持高中低三种可选的代码复杂度插桩渲染。
- **真实大模型直连**：本地 LocalStorage 安全存储用户提供的 `API Key` / `Base URL` / `Model`，原生直连生成真实数据。**支持基础、思考、多模态、生图等 4 组预置配置供快速切换。** 为确保预览紧凑性，预设按钮在空间不足时优先舍去图标保留文字。
- **深度思考 (CoT) 完美折叠展示**：支持解析像 DeepSeek-R1 输出的 `delta.reasoning_content` 数据，包裹进原生 `<details>`，流式持续输出的过程中用户反复展开/折叠的交互状态不受污染与重置（状态强追踪注入）。
- **原生多模态交互**：
  - **白嫖原法语音**：利用浏览器 `Web Speech API` 实现精准的免费录音级长文本识别，带实时呼吸动画。
  - **多模态发图**：支持上传本地图片直接解析为横排 Base64 缩略图（支持跨轮次的历史图 Token 精简），使用标准 VLM 数据结构向后传输。
- **真·多会话并行与组件隔离**：彻底解耦多会话管理。A 聊天室生成思维链的同时，切入 B 聊天室发问，两者的缓冲区记录、`AbortController` （中断句柄）、气泡状态均相互隔离且同时运作，完全匹配顶端商业 AI 级别体验。
- **状态感知重构**：发送按钮在响应中精准切转 Icon 与提示语，支持打字流期间打断 (Stop) 及渲染 Markdown （`marked`+`DOMPurify`组合）特性。TTFT （首字渲染前）伴有骨架波浪呼吸点动画。
- **Markdown 渲染优化**：修复了因二次排版（首行缩进）导致 Markdown 标题和列表无法正确解析的问题。现已自动识别并跳过干扰排版。

## 5. 存储与作品管理系统
- **物理量化存储看板**：存储管理区分 **作品总体积**（用户手动导入的文本）与 **缓存的总体积**（系统示例对话、骨架占位数据等），并提供一键格式化（彻底抹除本地 IndexedDB 与 LocalStorage）功能。
- **纯净级列表管理**：作品文件管理页仅响应手动加载的文件列表，排除所有由 AI 生成模拟或系统内置的“干扰项”，确保书架核心区的操作精准度。
- **智能布局对齐**：设置页内部调整高级功能层级，将“作品管理”与“存储看板”直接相邻挂载，强化数据管理的闭环体验。
- **对话类型分类逻辑**：引入显式的 `type` 字段（`novel` | `dummy` | `ai`）取代原有的文件名对等匹配逻辑，增强了数据分类的健壮性。
- **默认作品引导**：当用户首次加载且作品列表为空时，会自动从 `public` 文件夹静默导入内置作品《海底两万里》（示例作品）.txt，提升内容感知度。
## 6. 开发与自动化脚本
- **卡密生成脚本**：在 `package.json` 中集成了卡密生成工具：
  - `npm run generate-keys`：默认生成 10 个新卡密并存入 `cardkeys.json`，同时更新 `new_keys_export.txt`。
  - `npm run generate-one-key`：生成 1 个新卡密。
  - 生成逻辑底层由 `generateKeys.js` 驱动，支持自定义数量。

## 7. Dev 专属工具与测试
- **邀请功能模拟**：在 `dev` 模式下，支持在左下角阅读界面额外显示一个 "模拟新用户+1" 按钮，一键模拟设备接受自身邀请码并完成验证流程，方便快速调试邀请机制与解锁 Pro 流程。
