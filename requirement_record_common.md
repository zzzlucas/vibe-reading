# 核心理念及术语规范
**项目的核心竞争力：装饰成上班时查阅文档和执行 AI 任务，实际则为本地解析和渲染小说的沉浸阅读网站。它的伪装细节必须逼真。**

- **禁用敏感词**：项目代码和 UI 中**不要直接出现“伪装”这个词**，统一使用**“装饰”、“氛围感”、“极客皮肤”**等词语代替。
- **术语替换**：
  - 用 **“BOSS”** 代替 “老板”（如 BOSS 键，BOSS 模式）。
  - 用 **“作品”** 代替 “小说”。
- **避免侵权或穿帮**：当高度仿制大陆常见网站风格时，请使用替代词而非原始 Logo 和名称（例如将“博客园”改称为 **“经典博客”**）。

# 简单需求与体验微调 (Simple Requirements)
> 记录项目中功能逻辑简单、快速修改的界面表现及交互需求。依据不同的功能模块分类如下：

## 1. 样式与视觉微调 (Style Tweaks)
- **深色模式 (Dark Mode)**：去除蓝光、降低高对比透明度，确保经典博客和经典文档风格下的护眼及专业感。
- **边框与修饰**：移除 ChatMessage 中多余的页面导航区顶部虚线边框；高级 Boss 键的输入框和下拉框增加悬停和聚焦的阴影及高级边框样式。
- **文案与图标**：去除设置页多余的“打字机速度”等无关图标，确保布局精简；小说展示时确保能够正确显示当前阅读的书名 (Work Title)。
- **ChatGPT UI 优化**：将原有的 ChatGPT 主题优化为更接近原生 UI 的设计，包括取消 AI 消息头像占据空间、将底部输入框重构为药丸状样式、并精简右侧多余输入菜单以便深度还原环境。同时针对用户反馈的文字颜色偏紫感，对 ChatGPT 皮肤下的 `--text-muted` 和部分边框色值进行了去色（Neutralize）处理，迁移至更纯正的工业灰色系。
- **侧边栏表现与滚动固定 (Sticky Sidebar)**：针对 Gemini 和 ChatGPT 皮肤极简环境，重新优化了右侧工作区伪装大纲（Boss Sidebar）的悬停视觉和背景边框，使其融合度更高。并通过结构调整配合 `position: sticky` 属性对其纵向位置进行约束固定，确保该功能区侧边栏能够在用户进行超长正文滚动阅读时，始终停靠在右侧可视区域不被推出屏幕外，从空间维系了极客“工作界面”的稳定性。
- **首页文案随机化**：ChatGPT 极简首页支持在刷新时随机从 4 条预设问候语中抽取展示，增加交互新鲜感。
- **对比度修复**：修复了 ChatGPT 暗色模式下，输入框由于 Focus/Hover 色值定义错误导致的与背景色重叠（消失）的问题。
- **Favicon 动态与极简风格一致性**：对网站原有的 `pwa-512.svg` 图标进行大改重制，替换成了项目标志性的“四角星芒（Sparkle）”极简透明矢量图形。并在全生命周期的主 `App.vue` 环境中挂载监听器，使标签页的 favicon 能够实时跟随当前的皮肤风格动态热切换（例如 ChatGPT 风格会自动切换成 OpenAI Logo，回到 Gemini 或其他不支持定制图标的风格会自动回退使用这枚星芒原生透明 Icon），极大增强了“伪装”切换的细节真实感与视觉统一性。
- **ChatGPT 侧边栏按钮调整**：在 ChatGPT 风格的侧边栏中，去除了底部的“图片”、“Codex”和“项目”三个原版拟态按钮，替换成了更符合实际功能的“探索内容”和“使用说明”按钮，以便于用户快速访问荒原探索及系统帮助弹窗。

## 2. 交互与通知反馈 (Interaction & Toasts)
- **快捷跳转修复**：点击 "Quick Config" Toast 通知时，必须准确自动跳转锚定到设置中的 "排版配置" 二级模态框并正常开启。
- **通知成就 (Achievement Toasts)**："Boss Care" 成就需要样式低调（使用 Info 类型消除夸张 emoji）；而点击页脚幽默彩蛋必须有专属的高级 Achievement 通知表现形式。
- **控制台彩蛋 (Console Quotes)**：打开 Console (F12) 期间随机打印隐藏的预设趣味文案，建立个性极客感。
- **分享持久留存**：生成的邀请链接需在邀请界面上一直持久高亮显示，避免关闭面板丢失。
- **自定义工具快捷键幽默交互**：将输入框左侧“工具”按钮的单一“敬请期待” Toast 升级为带有操作按钮的 Toast（“将此按钮配置为其他功能快捷键”）。由于项目目前没有其他真正的快捷键功能，故点击后将直接跳转至“排版配置”设置面板，在界面和体验上达到所谓“选项只有一个‘排版设置’”的预期幽默效果。

## 3. 开发环境与辅助功能 (Dev & TestTools)
- **一键模拟邀请**：仅限 `dev` 开发环境下，在左下角阅读界面提供一个 "模拟新用户+1" 按钮快速进行设备被邀请后的状态模拟，以便于快速独立测试 “邀请 3 人解锁 Pro” 链路的闭环流转。

## 4. 路由与存储控制 (Routing & Storage)
- **子路由 SPA 回退拦截**：修复在子路由（如 `/app/...` 环境下，清理 localStorage 后重启引发的找不到本地 `.txt` 背景小说，进而致使 SPA 默认 fallback 并解析渲染了 `index.html` 的问题。将默认文案请求变更为根目录绝对路径以规避。
- **缓存前缀统一重命名**：将所有缓存使用的前缀从 `deep_reader` 统一改为 `find_deep`，确保项目内部标识符的一致性。

## 5. 部署与环境集成 (Deployment)
- **Vercel 部署支持**：确认项目当前的配置（如 `vercel.json`、`api/index.js` 及前端打包产物）已经完善，支持代码推送到 GitHub 后，直接在 Vercel 平台上实现“开箱即用”的前后端一键无缝部署。
- **重新罗列 Vercel 额度**：根据最新的控制台监控面板，在 `README.md` 中重新罗列了各项免费额度指标（如 Function Invocations 1M、Data Transfer 100GB 等），以提供更准确的资源与计费红线参考。

## 6. 用户文档与帮助说明 (User Manual)
- **使用说明文档与提示 (User Manual & Tooltips)**：编写并优化了一份面向用户的使用说明文件 `read_help.md`。核心内容明确了：① **无需注册登录**，采用本地存储机制；② **不提供任何预置资源**，由用户加载本地 `.txt` 及配置私有 AI 接口；③ 支持**基础、深度、识图、文生图**等多模态 AI 接入配置；④ 统一遵循项目的“氛围装饰”及“作品”术语规范。此文档在 `HelpModal.vue` 中动态解析并呈现，作为用户理解极客沉浸式阅读的核心指控中心。

## 7. 邀请裂变逻辑 (Invitation Logic)
- **优化同IP邀请限制**：将“同IP地址下最多允许建立受邀记录次数”从 3 次下调为 2 次，提示文案“当前网络环境限制”仍由尝试绑定的受邀新设备（超出次数的第3个及以上设备）触发查看。此改动旨在促使用户进行更广泛的跨网络/跨地理位置的裂变传播，提升单个邀请链接的传播价值。
- **防止 Postman 接口滥用 (Anti-Bot Validation)**：由于本项目非重度账号体系，没有传统登录 token，为了防止作弊者通过 Postman 或脚本批量伪造 `validate` 接口调用，已在 `/api/invite/validate` 接口加入了双端加盐哈希校验：前端必须传输实际经过的 `activeTime`（大于180秒），并使用项目中防篡改字数统计所使用的的同款 UUID 哈希算法 `_generateSecurityHash` 进行签名计算后一并传输给后端。后端复盘校验签名一致性才予放行激活，从而封堵了通过工具裸发 POST 请求直接白嫖激活奖励的漏洞。
- **全局 API 请求签名防护 (Global API Signature Middleware)**：在 `server.js` 中增加了全局中间件，对所有 `/api/` 路由进行签名校验（仅生产环境生效）。前端新增 `src/utils/request.ts` 统一封装 `apiFetch` 函数，调用所有 `/api/` 接口时自动注入三个请求头 `X-Device-Id`（设备ID）、`X-Timestamp`（毫秒时间戳）、`X-Signature`（由时间戳+路径+设备ID三者组合加盐哈希计算）。后端中间件验证：① 三个请求头是否存在；② 时间戳是否在90秒允许窗口内（防重放攻击）；③ 签名是否与服务端重新计算一致。所有 `/api/` 调用已从原生 `fetch` 统一迁移至 `apiFetch`。

## 8. AI 接入与配置 (AI Configuration)
- **多接口配置支持**：将原先单条固定 AI 模型配置重构为最多支持 5 条的多配置方案。用户可在设置中通过标签页交互自由增删、切换，并为每个接口设置专属的展示名称。主聊天页面输入框右下角新增配置切换菜单，实现交互过程中零阻力免打扰切换模型及接口源。
- **对话独立模型记忆**：系统支持每一个对话标签页单独记录并保持最近使用过的 AI 配置（通过将 ID 挂载至 Novel Meta）。若当切回对话时原数据源被删除，则智能降级降级为全局激活项或列表第一项，增强了沉浸感与体验连贯性。
